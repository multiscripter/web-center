1. Создать базу данных с 2 таблицами (товары, логи).

"Товары":
Поля: Идентификатор, Наименование, Цена, Доступное количество.
Заполнить таблицу не менее чем 10 товарами. 
При формировании полей товара использовать адекватные значения.

"Логи":
Поля: Поисковый запрос, кол-во найденных товаров, дата/время  запроса.

2. Сверстать веб-страницу и разместить на ней форму поиска товаров по названию.
Реализовать подгрузку списка подходящих товаров в асинхронном режиме (AJAX), т.е. при заполнении формы и нажатии кнопки "Найти" не должно происходить полной перезагрузки страницы.

Взаимодействие с пользователем:
Посетитель страницы набирает в поле для поиска название товара или часть названия товара, который он хочет найти.
При совпадении имени товара с поисковым запросом (или вхождении поискового запроса в строку имени товара),а также при условии не нулевого доступного количества, этот товар выводится (наименование, доступное кол-во, цена).
В строке "Наименование" цветом должно быть выделено совпадение с поисковым запросом.
При разработке функционала работы с БД  желательно использовать ООП (создать класс).

3. Необходимо записывать в лог файл (напр. log.txt) информацию о каждом поисковом запросе в формате: "поисковый запрос";"кол-во  найденных товаров";"дата выполнения запроса"
Каждый запрос это новая строка.

4. Создать на странице кнопку "Записать логи в базу". 
При нажатии на эту кнопку, информация из лог файла должна записаться  в БД, в таблицу "Логи".

Сам файл по завершению должен очиститься.
